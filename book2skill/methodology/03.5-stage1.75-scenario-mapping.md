# 阶段 1.75 — 真实场景映射

## 目标

把通过三重验证的方法论单元,映射到**现代生活中的真实问题场景**。

## 为什么需要这一步

书中的案例可能是几十年前的、特定行业的、或者作者个人的。
直接用这些案例做 A1 段,用户会感觉"和我无关"。

这一步的作用是:**把书中的方法论翻译成用户能感同身受的现代场景**。

## 执行流程

对每个通过验证的单元:

### 步骤 1 — 提取核心机制

问自己:这个方法论的本质是什么?用一句话概括。

示例:
- "逆向思维"的核心机制:先问"什么会让我失败",避免失败因素后反向推出行动
- "能力圈"的核心机制:只在自己能做出比市场更准判断的范围内做决策

### 步骤 2 — 列举现代场景

在当前时代的生活中,什么问题能用这个机制解决?

**场景类型** (尽量覆盖多种):
- **职场场景**: 面试、汇报、冲突、晋升、跳槽、创业
- **个人场景**: 决策、学习、关系、健康、理财
- **商业场景**: 产品、营销、融资、竞争、团队管理

每个类型至少写 1 个场景,总共 3-5 个。

### 步骤 3 — 写用户原话

遇到这些问题的用户会怎么说?

**好的用户原话**:
- "我想创业但怕失败,该怎么想这个问题"
- "怎么做才能让这次面试成功"
- "我纠结要不要接这个新项目,列了一堆好处但还是没底"

**坏的用户原话**:
- "我需要一个决策框架" ← 太专业
- "请帮我分析这个问题" ← 太抽象

## 产出格式

```yaml
- id: f01
  title: 逆向思维
  modern_scenarios:
    - scenario: "纠结要不要辞职创业"
      user_voice: "我想创业但怕失败,该怎么想这个问题"
      core_mechanism: "先问什么会让我失败,避免失败因素"
    - scenario: "准备重要面试"
      user_voice: "怎么做才能让这次面试成功"
      core_mechanism: "先问什么会让面试失败,避免这些因素"
    - scenario: "决定要不要接新项目"
      user_voice: "列了一堆好处但还是没底"
      core_mechanism: "反过来问最不希望发生什么"
    - scenario: "准备重要汇报"
      user_voice: "这个汇报很重要,我该怎么准备"
      core_mechanism: "先想什么会让汇报失败,避免这些因素"
```

## 质量门

- [ ] 每个单元至少有 3 个现代场景
- [ ] 每个场景都有对应的"用户原话"
- [ ] 场景覆盖不同类型 (职场/个人/商业)
- [ ] 用户原话读起来像真实用户会说的话

## 常见失败模式

1. **场景太抽象** — "用户需要做决策时" 不是场景,"用户纠结要不要辞职创业" 才是。
2. **用户原话太专业** — "我需要一个逆向思维框架" 不是用户会说的话,"我纠结要不要...但又怕..." 才是。
3. **场景类型单一** — 如果所有场景都是投资相关,说明你没有把这个方法论泛化。
4. **跳过这一步** — 直接用书中的老案例,用户会感觉"和我无关"。
